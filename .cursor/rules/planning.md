# Planning Rules

Before implementing any feature or making significant changes:

1. You **MUST** create and save a planning document inside the `Obsidian/` folder.
2. The plan document must include:
   - Feature name
   - Description
   - Technical approach
   - Tasks checklist

Commit messages must reference the plan when relevant.
Never start modifying code without first adding the planning document to the Obsidian directory.
